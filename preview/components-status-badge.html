Anatomy & rules

Shell: shares the Role Pill — padding: 3px 10px, border-radius: 999px, 500 12px Inter, 6px dot, gap: 6px; the pill hugs its content. sm = 11px / 1px 8px. The borderless variant drops the fill and keeps the dot + colored label.

Two forms. The soft pill is the default everywhere. The borderless form (dot + colored label, no fill) is only for rows that already carry a Role or Category pill, so the two don't read as twins.

Tone = meaning, not feature. Map a status to the nearest of the six tones (Neutral · Info · Pending/amber · Success/green · At-risk/orange · Danger/red) — never invent a new color per feature. Pending (amber) and At-risk (orange) are deliberately distinct hues so “awaiting” never reads as “problem”.

Attention is built in. There are no emphasis levels to choose — loudness comes from the tone itself (amber leans in, gray recedes) plus the live, pulsing dot, which is reserved for the single genuinely in-progress state and respects prefers-reduced-motion.

Status vs. other pills: a status describes a lifecycle state (it changes over time); Role/Category pills are fixed attributes. The dot is the tell — if a row already shows a Role pill, use the borderless variant for status.

Copy: one or two words, Title Case. No icons by default — color + dot carry the meaning, matching the system’s “color conveys status” rule.
