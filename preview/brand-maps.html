Maps

Any time the portal needs a map — routes, account locations, territory planning — use Leaflet with the CARTO Light All basemap. Two pin styles, one route stroke, a fixed attribution treatment. Never satellite, never colorful tiles.

  • Pins are Leaflet divIcons — render <div class="g-map-pin"><div>1</div></div> and pass className: 'g-map-pin-icon' to zero out Leaflet's default marker chrome.
  • Add .is-active to the currently selected / in-focus stop — inverted blue fill + halo. Never more than one active pin per map.
  • Add .is-unassigned for stops that haven't been slotted into a route yet. These carry no number; the red ring is the "needs attention" signal.
  • Label font is Inter Bold — don't substitute. Default label color matches the ring (#282838); the active variant flips it to white.

  • Default Leaflet attribution control, restyled: rgba(255,255,255,.8) background strip, 10px Inter, #555 text, primary-blue links.
  • Never remove attribution — OSM + CARTO both require it.
Usage
  • Load Leaflet 1.9.x CSS + JS from unpkg. Include maps.css after leaflet.css.
  • Tile URL: https://{s}.basemaps.cartocdn.com/light_all/{z}/{x}/{y}{r}.png — the {r} gets @2x on retina automatically.
  • Wrap the map container in <div class="g-map"> to pick up border, radius, elevation, and zoom-control + attribution restyling.
  • When embedding inline on a page (not a floating popover), add .is-inline to drop the shadow and soften the radius.
