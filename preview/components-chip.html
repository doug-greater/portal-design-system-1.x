Chip vs Status Badge vs Pill: Chip = micro flag / marker (icon, no dot, 10.5px). Status Badge = lifecycle state (dot, no icon, 12px). Pill = category / role tag (12px, no dot / icon).

Tone = meaning — map to the nearest of the six tones (neutral / info / amber / atrisk / danger / success); never mint a per-feature color. Copy is one to three words; the leading icon is optional.

atrisk (--g-orange-10 / --p-atrisk-strong) is a soft orange between amber and danger — "At Risk" / "~N draining". It's the chip-tint sibling of the saturated Inventory Condition swatch --cond-at (#F59E0B) used on the map/legend.

Chip Toggle is the interactive sibling — a boolean attribute pill (off: gray outline; on: --p-primary-tint + --p-primary-ink). Distinct from the 32px Filter Chip.

iconRight (1.7) renders a 12px trailing glyph after the label (mirrors the leading icon). Use a trailing directional arrow when a chip's job is to point at the control it recommends — the "Suggested arrow_forward" nudge now points toward the toggle it wants enabled (it dropped the leading lightbulb). Also carries entity glyphs on deep-link chips (POD Plans blur_medium, Store Promos award_star).
