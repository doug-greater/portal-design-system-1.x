Role-derived RBAC (1.8). One role → capability matrix is the source of truth, edited only here; saving re-syncs every user holding the role. The user page is read-only about permissions. (Supersedes the 1.5 per-user editor.)

Masonry packing (column-count: 2; every card break-inside: avoid) lets variable-height section cards interleave. The same matrix renders read-only (static checks) on Users → Access.

Amber = pending edit — a toggled-but-unsaved row tints --g-gold-10; the header shows an amber "Unsaved changes" pill, and a per-role 7px --p-warning dot (see Executive) marks any role with an unsaved draft so switching never hides pending edits. Save opens a confirm: "Update Department Manager permissions?" → "Apply to 12 users".
